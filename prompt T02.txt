1. Data stok barang disimpan dalam sebuah rak gudang yang tersusun rapi. Perhatikan pola penyimpanan pada gambar di bawah ini untuk memahami bagaimana kelompok barang dipisahkan. Input:
Jumlah total data (N).
Deret stok (s1, s2, ..., sN).
Kode kategori barang.

Tugas Buatlah program yang mampu menentukan total stok hanya untuk kategori barang yang diminta.

"Bertindaklah sebagai seorang Mahasiswa semester 4 it del. Tugas Anda adalah merancang dan mengimplementasikan sebuah modul logika. Sistem ini bertujuan untuk mengotomatisasi pengelolaan inventaris di gudang pusat yang selama ini masih dilakukan secara manual, sehingga sering terjadi kesalahan pelacakan jumlah stok antar kategori barang. Saya ingin Anda menjelaskan logika berpikir dan langkah-langkah pemecahan masalah (step-by-step reasoning) sebelum menuliskan kode."
1. Konteks & Alur Kerja Sistem:
Tujuan Spesifik: Membangun mesin pencari stok (stock aggregator) yang mampu memfilter data dari urutan fisik rak gudang berdasarkan kode kategori tertentu.
Alur Kerja: 1. Sistem menerima input berupa jumlah total barang ($N$).
2. Admin memasukkan deret nilai stok ($s_1, s_2, ..., s_N$).
3. Admin memasukkan kode kategori target.
4. Sistem melakukan iterasi, memvalidasi kategori, dan menjumlahkan stok yang relevan.
5. Sistem menampilkan total akhir secara presisi.
2. Spesifikasi Teknis (Kontekstualisasi):
Bahasa Pemrograman: Java (versi 11 atau terbaru).
Lingkungan: Aplikasi Konsol berbasis teks (CLI).
Library: Hanya menggunakan library standar Java (java.util.Scanner untuk input).
Platform: Desktop (Windows/Linux terminal).
3. Struktur & Format:
Organisasi Kode: Gunakan pendekatan Object-Oriented Programming (OOP). Pisahkan logika menjadi minimal dua bagian: Kelas WarehouseManager (untuk pemrosesan data) dan kelas Main (untuk interaksi pengguna).
Format Input: Input diberikan melalui terminal secara berurutan ($N$, lalu deret angka, lalu kode kategori).
Format Output: Output berupa teks tunggal yang merepresentasikan total stok (Integer).
Cakupan: Berikan kode lengkap yang siap dijalankan (ready-to-run).
4. Keterbatasan & Standar Kualitas:
Validasi: Pastikan ada validasi input untuk memastikan $N$ tidak negatif dan data stok yang dimasukkan adalah angka valid.
Clean Code: Gunakan penamaan variabel yang deskriptif (misalnya totalStock, categoryCode), tambahkan komentar pada bagian logika pemisahan stok, dan pastikan kode mengikuti standar Java Naming Convention.
5. Contoh Interaksi (Input/Output):
User Input:
N: 5
Deret Stok: 10 20 15 10 30
Kategori Target: 10
Expected Output:
Total stok untuk kategori 10 adalah: 20
(Penjelasan: Sistem menemukan dua item dengan nilai 10 dan menjumlahkannya).
Instruksi Tambahan untuk AI:
"Mohon analisis terlebih dahulu bagaimana pola penyimpanan di rak gudang mempengaruhi cara pembacaan array sebelum Anda menuliskan fungsinya. Pastikan kode efisien dengan kompleksitas waktu $O(N)$."

Pada Eclipse Java Project (EJP), source code akan diletakkan pada direktori ```src```, sementara hasil kompilasi (bytecode) diletakkan pada direktori ```bin```. Anda hanya diperkenankan untuk memodifikasi berkas dalam direktori ```src```. Modifikasi pada berkas lainnya dapat berakibat pada kegagalan proses penilaian.

## 4. Bagaimana penilaian akan dilakukan?
Penilaian dilakukan dengan memperhatikan aspek-aspek berikut:
1. Solution correctness. Ketepatan solusi akan dinilai secara otomatis dan objektif oleh GitHub Classroom (GHC). Pendekatan black-box yang memperhatikan kesesuaian keluaran dengan yang semestinya. Dalam memeriksa, skema yang digunakan adalah exact output artinya solusi anda harus memberikan keluaran yang sama persis dengan expectation.
2. Integrity check. Pekerjaan anda juga akan diperiksa tingkat kemiripannya dengan pekerjaan rekan anda dengan menggunakan [JPlag](http://jplag.ipd.kit.edu/).
